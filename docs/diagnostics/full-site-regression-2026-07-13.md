## 可重复基线

- `npm run check`：修复前既有 5 个课表测试通过。
- `npm audit --omit=dev`：已知生产依赖漏洞为 0。
- 课表历史文本节次经共享核心解析后必须得到 1-12 的有限整数位置。
- 线上浏览器连接检查连续超时，因此不把该现象算作站点缺陷，改用本地 HTTP 与部署文件证据。

## 排序后的假设

1. **认证响应缓存造成数据旧或账号间观感不一致。** Cloudflare 代理对若干登录态 GET 返回 `private, max-age`；可由源代码与响应头断言验证。
2. **上游错误未脱敏造成信息泄露。** Node 对非数据库 500 返回原始 `error.message`，边缘代理返回 `detail`；可用构造错误和静态断言稳定复现。
3. **长期运行变慢与限流 Map 无界增长有关。** 四个进程级 Map 没有统一清理；可通过写入大量键并观察容量验证。
4. **首屏慢主要来自单体资源而非数据库。** `app.js` 与主样式为单体加载；需要资源体积和路由加载时序证据，当前记为结构性性能债务。
5. **课表下沉来自非法 CSS 网格位置。** 既有红灯用例已证实并由共享解析核心修复，本轮继续作为回归门禁。

## 诊断循环状态

- 第一轮新增测试先预期失败：缺少安全策略模块、代理仍泄露 `detail`、CSP 与运行时代码不一致、非法 JSON 仍可能成为 500。
- 修复后 `npm test` 为 12/12，通过课表定位、安全策略、HTTP 冒烟、Excel 导入、四角色筛选和分页测试。

## 已确认根因与修复

1. **身份库筛选与翻页偶发失效。** 页面同时使用浏览器缓存、页缓存和“角色统计为 0 即不请求”的推断，部署数据变化后可能返回旧页或假空页。现已统一设置 API `no-store`，删除页缓存和假空页捷径，每次筛选、搜索和翻页都读取服务端分页结果。
2. **课表课程卡片下沉。** 历史导入数据中的中文节次、空值和非法数值直接进入 CSS Grid。现由 `timetable-core-v155.js` 统一规范为 1-12 节和有效跨度，前端、服务端 Excel/OCR 导入复用同一算法。
3. **生产错误泄露内部信息。** Node 500 和 Cloudflare 代理 502 曾返回数据库或上游错误。现数据库错误映射为 503 通用提示，未知生产 5xx 脱敏，内部日志保留请求编号。
4. **长期运行内存增长。** 限流 Map 无容量边界。现改为有最大键数、最大事件数和过期清理的滑动窗口注册表。
5. **部署链路数据旧。** 登录态 GET 曾可被边缘缓存。现认证请求和前端 API 均 `no-store`，公开目录数据才允许短期缓存。

## 残余性能债务

- `public/app.js` 和主样式仍为大单体，后续应按路由拆分并延迟加载 AI、软件库、综测等模块。
- 青春主题图片与音乐资源较大；音乐已 `preload=none`，图片仍建议生成 WebP/AVIF 多尺寸并按视口选择。
- Cloudflare Pages 到 Vercel 再到 TiDB 存在额外网络跳数；数据库应选择香港或邻近区域，并保持连接池与平台代理超时配置一致。
