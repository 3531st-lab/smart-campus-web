## 风险清单

### 高风险

1. **浏览器持久保存认证令牌。** 当前令牌位于 `localStorage`，一旦发生 XSS 可被读取。建议迁移为 `Secure; HttpOnly; SameSite=Lax/Strict` Cookie，并增加服务端撤销/轮换机制；迁移后同步增加 CSRF 防护。
2. **个人 AI API Key 位于浏览器存储。** 第三方模型密钥属于高价值凭据，建议默认仅保存在会话内，或使用服务端加密凭据库并让用户可撤销；不得写入日志、分析平台或前端错误上报。
3. **生产数据库与备份证据不足。** 此前出现 TiDB `Access denied`，代码现已脱敏但不能替代正确的账号授权、IP/网络策略、最小权限和备份恢复演练。发布后必须验证读写与故障恢复。

### 中风险

1. CSP 因现有大量内联样式仍包含 `style-src 'unsafe-inline'`。建议样式模块化后改用 nonce/hash 或外部样式表。
2. 管理端 AI 运行配置可写入本机 JSON，密钥为明文。生产环境应只使用平台 Secret/加密密钥库，并禁用文件落盘。
3. 审计日志虽记录关键管理动作，但尚无不可篡改存储、统一留存期、检索告警和时间同步证据。
4. 总管理员尚未强制多因素认证；高权限操作缺少二次确认与短时重新认证。
5. 静态无状态令牌在有效期内难以单独撤销；账号停用后应立即拒绝，并建议增加令牌版本或撤销表。
6. 数据删除、更正、导出、保存期限和第三方处理者清单尚未形成完整闭环。

### 低风险与性能项

1. 前端脚本和样式仍为单体，首屏解析与后续维护成本偏高。
2. 主题图片和音乐较大，应转 WebP/AVIF、提供响应式尺寸并继续保持音频延迟加载。
3. Cloudflare Pages → Vercel → TiDB 的跨平台链路增加延迟和故障点，应通过区域一致、连接池、可观测性和适当的公开数据边缘缓存优化。

## 整改优先级

### 发布前必须完成

- 在 Vercel 配置不少于 32 字符的 `AUTH_SECRET`，并配置独立的短信、数据库和第三方服务 Secret。
- 用最小权限数据库账号验证身份库导入、四角色筛选、翻页、课表读写和实验室预约审批。
- 确认 Cloudflare Pages 代理的 `API_ORIGIN` 指向当前 Vercel 生产地址，认证响应为 `no-store`。
- 启用 TiDB 自动备份，执行一次恢复到隔离环境的演练并保存记录。

### 30 天内

- 迁移认证令牌到 HttpOnly Cookie，并建设撤销/轮换能力。
- 迁移个人与管理员 AI 密钥到加密凭据存储。
- 为总管理员启用 MFA、敏感操作二次确认和安全告警。
- 接入 Sentry 或同类平台时启用个人信息脱敏，禁止采集手机号、令牌和 API Key。
- 建立安全事件、漏洞、备份恢复、账号复核和日志留存制度。

### 90 天内

- 完成定级论证、备案准备和有资质机构的正式测评。
- 完成前端路由拆分、资源压缩与性能预算。
- 建立个人信息处理活动清单、第三方委托协议、保存期限和用户权利响应流程。
